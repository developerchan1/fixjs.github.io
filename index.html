Thinking Big and Working Collaboratively

For the Web to compete with native platforms, I believe we have to think big. This means building on our competitive strengths like URLs and dynamic loading, as well as taking a hard look at our platform’s weaknesses — lack of access to modern hardware, failures of the offline experience, or limitations of cross-origin communication, to name a few. --Dave Herman

This is what we are concerned with in FixJS as a team of JavaScript enthusiasts.

Promoting and Writing Modular JavaScript

For JavaScript to make it easier to write clean and maintainable code. What we are working on these day, in this specific regard is mostly dealing with JavaScript module patterns and coding styles to come up with a lightweight, effective and efficient approach.

So far we have come up with a lightweight implementation of AMD module format which still doesn't support node modules yet, but it will soon do it.

DefineJS

A JavaScript module loader, an implementation of AMD (Asynchronous Module Definition).
